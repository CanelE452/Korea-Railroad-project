# 2026-05-12 — 실시간 추론 False Positive 대응

## 배경

`scripts/dope/run_dope_live.py` (RealSense D435i) 실시간 데모에서 팔레트가
없는 장면에도 매 프레임 검출이 발생 (사용자 보고). 코드 진단 결과 학습 측
문제(F5 real-only over-fit) 이전에 inference 코드 자체에 두 가지 직접 원인
존재.

## 원인 1: Raw-peak Fallback 블록

`run_dope_live.py:365-404` 의 fallback 로직:

- DOPE 정식 감지(`ObjectDetector.find_object_poses`) 실패 시 9 채널 belief
  map 중 `threshold` 를 넘는 raw peak 가 4 개 이상이기만 하면 `detected=True`
  처리 + 가짜 2D yaw 화살표 표시
- DOPE belief map 은 "객체 없음" 클래스가 없어 항상 어딘가에서 peak 가 생김
  → 사실상 "항상 검출됨" 효과
- 정식 6D yaw 추정과 무관한 raw-peak 기반 2D yaw 만 보여주므로 디버그 가치
  대비 false-positive 비용이 큼

## 원인 2: 기본 threshold 가 너무 낮음

- `--threshold` default 0.01 (belief peak 1% 만 넘어도 keypoint 인정)
- `thresh_map` 0.001, `thresh_points` 0.001 — DOPE detector 내부 후보
  생성 임계값도 사실상 0
- Belief click auto-tune 의 `min * 0.7` 도 사용자가 클릭한 진짜 peak 의
  70% 까지 허용 → 너무 관대

## 변경

파일: `scripts/dope/run_dope_live.py`

### 1) Fallback 블록 제거

```
Before (365-404):    raw peak ≥ 4 → detected=True + 2D yaw 화살표
After:               제거. 정식 ObjectDetector.find_object_poses 결과만 신뢰
```

NOT DETECTED 표시(407-409 줄)는 그대로 유지 — 미검출 프레임이 명시적으로
드러남.

### 2) Threshold default 상향

```
파라미터              Before     After    슬라이더 max
─────────────────────────────────────────────────────
--threshold           0.01       0.10     100 → 300   (0.0~0.3)
thresh_map            0.001      0.05      50 → 200   (0.0~0.2)
thresh_points         0.001      0.05      50 → 200   (0.0~0.2)
auto_threshold 비율   min×0.7    min×1.0  —
auto-tune clip 범위   100/50/50  300/200/200
```

슬라이더 범위 확장으로 현장에서 0.3 까지 raise 가능. Auto-tune 도 사용자가
클릭한 가장 약한 peak 의 100% 를 cutoff 로 잡아 noise 침투 차단.
