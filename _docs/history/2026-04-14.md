## Metric 자체 검증 — NN matching pipeline 무결성 증명

### 배경
발표 방어 대비 — "metric을 4번이나 바꿨는데 지금 것이 맞다는 보장?" 질문 선제 대응.
근본 재실험(seed 5개, Seen/Unseen 평가)은 2일 내 물리적 불가 → pipeline 자체가 수학적으로 올바르다는 걸 합성 val 데이터로 증명.

### 추가 스크립트
- `scripts/data_prep/eval/validate_nn_metric.py` — 4 test 자동 실행
- 합성 val 3000장 중 500장 사용 (GT = 렌더링 camera projection, 완벽 정확)

### 결과 (500 frames × 8 corners = 4000 matches)

```
T1 Identity (GT=Pred):        max=0.00e+00, mean=0.00e+00   PASS
T2 Permutation (GT shuffled): max=0.00e+00, mean=0.00e+00   PASS
T3 Known noise (sigma=10px):  mean=12.53px, expected=12.53  PASS (오차 0.00px)
T4 Swap invariance (2 swap):  NN=0.00 vs Direct=48.54px     PASS
```

### 발표 방어 포인트
1. **T1/T2**: pipeline 자체 무결성 — 같은 점끼리 매칭 시 정확히 0
2. **T3**: 스케일 정확성 — Rayleigh 이론값과 이상적 일치
3. **T4**: **왜 direct index를 버렸는지 수치 증거** — 2-corner swap만으로도 48.5px 허위 오차.
   이전 ep65=5.0% (direct) → 21.6% (NN) 차이는 metric 버그가 아니라 direct index의 구조적 한계.
