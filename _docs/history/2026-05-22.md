## v5 작성 — face normal yaw + camera distance 기반 FRONT 식별

### 배경
v4 는 4 vertical face 의 image polygon area 비교로 FRONT 선택. 단점:
- projection + truncation 영향 — truncated FRONT face 가 area 작아 보일 수 있음
- 2D 평면 측정이라 face 의 실제 카메라 정합도를 못 반영

v5 는 3D pose 만 사용:
- 각 face 의 normal vector (outward, camera frame) 과 line-of-sight 의 yaw_alignment
- face center 의 camera 거리 (가까울수록 score 큼)
- 두 score 의 가중합: `score = α * yaw_align + β * proximity`

### 알고리즘 (v5)
1. v4 와 동일하게 top4/bot4 분리 + vertical pairing → 평행 split 2개 → 4 face 후보 (실패 시 None)
2. 각 face 에 대해 camera frame 에서:
   - face_center = mean of 4 corner
   - face_normal = cross(p1-p0, p2-p0) normalize. body_center→face_center 방향과 같은 쪽으로 부호 강제 (outward)
   - los = face_center / |face_center| (origin→face)
   - yaw_align = dot(face_normal, -los) ∈ [-1, 1]
3. distance_proximity: 4 face 의 거리를 min-max normalize 후 inverse (가까운 face=1, 먼 face=0)
4. score = α * yaw_align + β * proximity. α=0.7, β=0.3 (yaw 가 주)
5. score 최대 face = FRONT, 같은 split 의 짝 face = REAR
6. LR (image x), TB (origin z) 는 v4 와 동일

### Camera frame 변환 (v3 와 같은 logic 재사용)
- mixed_v8: pose_transform (4x4, object→camera) 로 cuboid 변환
- v1/v2: keypoints_3d_world + camera_data.{location_worldframe, quaternion_xyzw_worldframe} 로 world→camera
- camera convention (OpenCV vs USD 부호) 은 face_normal 의 outward 강제 + los unit 방향으로 자동 처리됨 (yaw_align 부호가 convention 무관)
