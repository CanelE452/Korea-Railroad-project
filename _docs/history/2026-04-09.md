## Visibility-Aware DOPE v1 구현

### 설계
- geometry-derived per-keypoint visibility (3단계: visible 1.0, self-occluded 0.5, out-of-frame 0.0)
- 인접 face 기준: 각 corner의 인접 cuboid face의 front-facing 여부로 판정
- centroid(8)는 별도: in-frame→1.0, out-of-frame→0.0
- coord_vis_loss: weighted average (가중치 합으로 정규화)
- weighted PnP: sigma-only, clip(0.3, 1.8) + normalize, peak gate

### 변경 파일
- `Deep_Object_Pose/common/utils.py`: _compute_visibility() 메서드 추가, batch에 visibility tensor 추가
- `Deep_Object_Pose/train/geo_loss.py`: coord_vis_loss(), VisibilityCoordLoss 클래스 추가
- `Deep_Object_Pose/train/train.py`: --vis_coord_loss/--vis_lambda/--vis_warmup 인자 + 통합
- `scripts/train_dope.sh`: --vis_coord_loss 전달

### 검증
- visibility GT 분포: visible 87%, self-occluded 4%, invisible 9% (20 sample)
- VisibilityCoordLoss 동작 확인

## 미완료 항목 구현

### 1. Conditional Incidence Filter (filter_D)
- `canonical_filters.py`에 `filter_D()` 추가
- 같은 face의 두 대각선 중점 거리를 projected diagonal로 normalize
- "있는 점들만으로" 검사 — 4개 corner가 다 있는 face만 체크, 없으면 pass
- `infer_and_filter_v2.py`, `infer_and_filter_v3.py`에 filter_D 통합
- v8_A noapril 결과: D pass 75/117 (64.1% of PnP)

### 2. Training Logging 개선
- `train.py`에 ratio 로깅 추가: `ratio_geo_bel`, `ratio_struct_bel`, `ratio_rel_bel`, `ratio_vis_bel`
- belief peak health 모니터링: `health/belief_peak_mean` 매 epoch TensorBoard 기록
- 보조 loss / base belief 비율이 20% 초과 시 과도 판단 가능

### 3. Weighted PnP 개선
- `pnp_solver.py`: weight = clip(1/(sigma+eps), w_min, w_max), mean=1 normalize
- gamma=1 (sigma 그대로, sigma^2 아님)
- w_min=0.3, w_max=1.8

### v8_vis 학습 완료

```
실험                     PnP    B     C    B∧C
──────────────────────────────────────────────────
1. v8                     93     1     0     0   baseline
2. v8 + wPnP              93     0     0     0   solver만 → 무효
3. v8_A                  117    16     6     6   ★ coord-only 최고
4. v8_A + wC             117    16     7     6   C +1
4. v8_A + wPnP           117     9     7     6   B↓
5. v8_vis                 71     9     3     1   vis-aware → PnP 하락
6. v8_vis + wPnP          71     5     3     3   B∧C +2
```

### 결론
- **v8_A (coord-only) 확정적 최고** (B∧C=6)
- v8_vis는 PnP 117→71로 크게 하락 — visibility weighting이 detection을 약화
- weighted PnP 단독(v8+wPnP)은 base가 약하면 무의미
- weighted C(v8_A+wC)는 소폭 개선 (C 6→7) — 유일하게 의미 있는 solver 개선
- visibility-aware supervision은 현재 설계에서 coord-only를 이기지 못함
