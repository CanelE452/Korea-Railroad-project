## Symmetric Loss 구현 (180° front-back swap)

### 배경
- 팔레트(991×1192mm)는 앞/뒤가 시각적으로 거의 동일
- 카메라 각도에 따라 0-3이 앞일 수도, 4-7이 앞일 수도 있음
- 기존 MSE loss는 둘을 다른 정답으로 취급 → belief map 혼란 → 납작한 cuboid

### 구현
- `Deep_Object_Pose/train/train.py`
  - `--symmetric_loss` CLI 플래그 추가
  - belief loss에서 원본 GT와 180° swap GT 중 min을 취함
  - swap 매핑: 0↔4, 1↔5, 2↔6, 3↔7, centroid(8) 유지
- `scripts/train_dope.sh` — `--symmetric_loss` 전달 지원

### 설계 결정
- 180° swap만 고려 (90° 회전은 991 vs 1192mm로 시각적 구분 가능)
- affinity field는 swap 안 함 (belief만 swap)
- trade-off: 앞/뒤 방향 구분 포기 (팔레트는 실용적으로 문제없음)

## 분석: Symmetric Loss 효과 재평가

Synthetic val에서는 잘 되고 (PnP 86%), real 밝은 팔레트에서도 잘 됨 (PnP 85%).
어두운 팔레트에서만 부족 (PnP 37%) → **swap 문제가 아니라 데이터 도메인 차이가 원인.**

```
근거:
  - swap이 원인이면 synthetic에서도 안 돼야 하는데, synthetic은 잘 됨
  - 팔레트는 형태가 단순해서 앞/뒤 구분 자체가 큰 문제가 아님
  - mixed_v7_sym의 개선은 symmetric loss + dark 데이터 + augmentation이 동시에 적용되어 개별 효과 분리 불가
  - 핵심 병목: 어두운 팔레트 학습 데이터 100장(전체 1.1%)으로 절대적 부족
```

**결론**: 어두운 팔레트 + 실제 환경 데이터를 더 만들거나 수집하는 것이 가장 필요.
