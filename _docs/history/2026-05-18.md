### 3. up.obj 정리 단계별 진행

#### 3.1 Loose parts 분리
- `bpy.ops.mesh.separate(type='LOOSE')` → 1088개 컴포넌트 분리
- Top 4 컴포넌트 face 수:
  - `up`: 16,928,591 (메인, 4.01×4.90×5.69)
  - `up.004`: 457,712 (잡동사니 1, 0.95×0.95×1.85)
  - `up.002`: 416,120 (잡동사니 2, 1.50×1.99×0.96)
  - `up.001`: 42,703 (잡동사니 3, 1.29×1.93×1.00)
- 나머지 1084개 작은 컴포넌트 → 삭제

#### 3.2 Decimate
- `up` (16.9M faces) → `Decimate modifier ratio=0.03` → 507,856 faces로 축소
- 통신 timeout 났지만 modifier는 실제로 apply됨 (vertex count 변경 확인)
- 잡동사니 3개도 함께 삭제 (Top view에서 색깔 구분 후 팔레트와 떨어진 별개 객체 확인)

#### 3.3 환경 cut
- **Z<0 cut**: `bpy.ops.mesh.bisect(plane_co=(0,0,0), plane_no=(0,0,1), clear_inner=True)`
  - 결과: 507K → 245K faces (창고 바닥 제거)
- **+Y noise cut**: `plane_co=(0,0.5,0), plane_no=(0,1,0), clear_outer=True`
  - 결과: 245K → 188K faces (윗쪽 너덜거림 제거)

#### 3.4 PCA 기반 자세 정렬 (실패)
- vertex coords에 PCA 적용 → 3개 주축 (eigenvectors) 추출
- 축 비율 결과:
  - mesh: 1.0 : 0.827 : 0.285
  - 실제: 1.0 : 0.846 : 0.085 (1.30 : 1.10 : 0.11)
- **문제**: 짧은 축이 실제보다 **3배 두꺼움**. 환경 노이즈가 두께 방향으로 늘어져서 PCA가 부정확.
- 정렬 후 Top view에서 데크보드 패턴 **안 보임** → 정렬 매트릭스가 데크보드 normal과 어긋남

### 4. 통합 전략 결정 (사용자 협의)

#### 4.1 사용자 입력 (핵심 결정)
- **목표**: 완벽한 좌우대칭 팔레트 OBJ 1개. 옆면 포크 슬롯 뚫림 보장.
- **베이스 선택**: up.obj가 윗면 깨끗 (단 아랫면은 바닥에 붙어 캡처 안 됨)
- **접근법**: 1/4 영역만 깨끗하게 추출 → Mirror X + Y로 전체 완성 (4면 entry 대칭)
- **현재 진행 방향**: 2 → 3 → 1 순서로 세 방식 결과 비교

#### 4.2 작업 옵션 정리
1. **Photogrammetry 정리 재시도**: face normal 기반 정렬, 1/4 추출, mirror. 30-60분, 결과 노이지 가능
2. **Procedural + 텍스처 베이크**: 정확한 1.10×1.30×0.11m 메시 생성 + laydown 텍스처 UV bake. 20분, 슬롯 완벽
3. **Hybrid**: 스캔의 깨끗한 데크보드 표면만 추출 → procedural 메시 위에 부착
