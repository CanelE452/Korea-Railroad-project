## 발표 스크립트 숫자 확정 — 3가지 metric 교차 검증

### 배경
발표 스크립트에 있던 "13.2% → 25.5% → 62.7% progression" 은 **전부 구 metric (PnP self-reproj) 기준**.
현행 NN matching 기준으로 재측정한 수치가 동일 모델에 대해 다른 값을 냄 → 확정 필요.

### 같은 두 모델 × 3가지 metric

```
metric                              ep60      ep65      상태
─────────────────────────────────────────────────────────────────────────────
(A) PnP self-reproj                 13.2%    25.5%     폐기 (self-referential)
(B) Raw kp vs GT, direct index       0.5%     5.0%     폐기 (convention 민감)
(C) NN matching (Hungarian) ★       18.9%    21.6%     현행 확정
```

### 중요한 발견 — metric 변경의 편향 없음

- ep60: 13.2% → 18.9% (+5.7pp 상승)
- ep65: 25.5% → 21.6% (**-3.9pp 하락**)

metric 변경이 한 방향 편향이 아니라 모델별로 다르게 움직임. "숫자 올리려고 metric 바꾼 게 아니라 구조적 결함 제거가 목적이었다"는 증거.

### 62.7% vs 60.5% 구분 확정

- **62.7%** = F4 실험 (RANSAC only 9PL) × 구 metric (per-keypoint or PnP reproj)
- **60.5%** = F5 실험 (RANSAC+LOO 2PL) × 현행 metric (NN matching)

둘은 다른 실험 + 다른 metric → 애초에 비교 불가. 발표에서 62.7%는 절대 언급 금지.

## Coord+Edge 가 NN matching 에서 좋아진 메커니즘 분석

### 왜 PnP rate 와 NN matching 이 반대 방향을 가리키나

```
Loss 역할 차이:
  Coord loss:   각 keypoint 의 절대 위치를 GT 에 가깝게
  Edge loss:    keypoint 쌍의 상대 거리 비율을 cuboid 구조에 맞게
                (0→1 edge / 0→4 edge ≈ width/depth 비율 유지)
```

Edge loss 는 **"8 corner 가 직육면체 구조를 유지"** 하게 만드는 regularizer.

### PnP rate 축 (구 primary metric)
- PnP 는 8 corner 전부 정확해야 풀림
- Edge loss 는 전체 구조 유지하지만 개별 corner 를 정확히 끌어오진 않음
- → "구조는 맞는데 개별 corner 가 몇 px 틀린" 상태 다수
- → PnP 가 reject → PnP rate 54.3% (coord only 62.2% 대비 -8pp)

### NN matching 축 (신 primary metric)
- 개별 corner 절대 위치만 봄 (Hungarian assignment)
- Edge loss 가 outlier corner (한쪽 면만 튀는) 억제
- → mean/median error 자체 감소

### 결정적 증거 — median shift
```
                   coord only       coord+edge       차이
──────────────────────────────────────────────────────────
median error       66.2px           16.7px           -49.5px
<20px              21.6%            38.4%            +16.8pp
Frames w/ pred     313/440          303/440          -10
```

Median 이 66.2 → 16.7 로 크게 감소 → **분포 전체 좌측 shift**.
평균만 움직인 게 아니라 median 이 움직였다는 건 edge loss 가 극단적 outlier
(한쪽 면 collapse 등) 를 효과적으로 잡았다는 강한 신호.

### 결론: 두 metric 모두 맞음, 다른 축을 잼
```
구 판단 (PnP rate):     coord only 가 optimum  ← 맞음 (PnP 기준)
신 판단 (NN matching):  coord+edge 가 더 정확    ← 맞음 (keypoint 기준)
```

당시 primary metric 이 PnP rate 였으므로 "coord only optimum" 결론은 합리적
결정이었음. NN matching 으로 전환하면서 관점 변화.

### ⚠️ 주의 — NN matching 의 관대함 리스크 (솔직 평가)

NN matching (Hungarian) 은 "구조만 맞고 corner 가 살짝 틀린" 경우도 낮은 cost
로 매칭 가능. 극단적으로 8 corner 가 전부 10px 평행이동해도 낮은 error.

→ coord+edge 38.4% 가 **"구조 보존 덕에 NN 이 관대하게 매칭한 결과"** 일
가능성 존재. 실제 pose 정확도는 PnP 기준이 더 엄격.
