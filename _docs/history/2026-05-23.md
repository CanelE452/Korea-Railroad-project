### 배경
사용자 결정 (5/22): **camera-facing convention** — 0~3 = 카메라 가까운 큰 면, 0=좌상, 1=우상, 2=우하, 3=좌하. TOP={0,1,5,4}, BOTTOM={3,2,6,7}.

학습 데이터 GT JSON (mixed_v8 + v1 + v2 = 28,991 file) 의 0~7 idx 를 위 컨벤션으로 permute 필요. dataloader 는 `projected_cuboid` 만 학습 target 으로 사용.

### 3D agent 위임 결정 → v4 한 번에 해결

3d-expert subagent 호출. 이 결정이 **돌파구**:

**v4 alogorithm** (`challenge/scripts/convert_to_camera_facing_v4.py`):
1. origin frame z 기준 top4 / bot4 split (3D gravity-up)
2. top↔bot vertical pairing (xy 거리)
3. top4 의 평행 split 2가지만 채택 (diagonal 제외)
4. **4 vertical face 후보 중 image polygon area 최대 = FRONT** (cam-frame 부호 의존 X)
5. parallel opposite = REAR
6. FRONT top-edge LR by image x, bot 은 vertical pair 자동
7. REAR 동일, centroid 그대로

**핵심 발견 — v3 실패 본질**:
- mixed_v8: OpenCV camera convention (cam.z > 0 = forward)
- v1/v2 (Isaac Sim USD): OpenGL/USD convention (cam.z < 0 = forward)
- v3 가 `cam.z 작은 = near` 가정 → v1/v2 에서 정확히 반대

v4 는 image polygon area 만 사용 → 데이터셋 차이 무관 + truncation 영향 X.
