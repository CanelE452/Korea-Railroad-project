## Geometric Loss 구현 (soft-argmax + BPnP)

### 신규 파일
- `Deep_Object_Pose/train/geo_loss.py` — Geometric Loss 모듈
  - `SpatialSoftArgmax2D`: belief map에서 미분 가능한 keypoint 추출
  - `BPnP`: cv2.solvePnP forward + implicit differentiation backward
  - `rodrigues_batch`, `project_points`, `projection_jacobian`: 미분 가능한 projection 유틸리티
  - 5가지 loss: kp_l2, diagonal, reprojection, volume, ADD
  - `GeometricLoss` 모듈: 전체 통합, warmup 지원, validity masking

### 수정 파일
- `Deep_Object_Pose/train/train.py`
  - `_runnetwork()`에 geo_loss_module, geo_lambda, geo_warmup 인자 추가
  - training loop에서 geometric loss 계산 및 합산
  - TensorBoard에 `loss/train_geo` 스칼라 기록
  - CLI 인자 추가: `--geo_loss`, `--geo_lambda`, `--geo_warmup`, `--geo_temperature`, `--geo_fx/fy/cx/cy`, `--geo_img_w/h`
- `scripts/train_dope.sh` — `--geo_loss`, `--geo_lambda`, `--geo_warmup` 전달 지원
- `_docs/models/training_loss.md` — Geometric Loss 섹션 추가 (구성, 파라미터, 사용법)

### 설계 결정
- `--geo_loss` 플래그 없으면 기존과 완전히 동일 (backward compatible)
- DOPE 모델 구조 변경 없음 — geo_loss는 loss 계산용으로만 사용, inference 시 제거
- BPnP warmup: 초기 epoch에서 belief 예측이 불안정할 때 PnP 실패 방지
- PnP 실패 시 validity mask로 해당 샘플 자동 skip

## Codex 분석: DOPE 성능 저하 원인

- 어두운 팔레트 실패: 80% 데이터/augmentation 문제, 20% geo loss 부작용
- Geo loss가 감지율 하락시키는 원인: soft-argmax가 강한 peak만 보상, 약한 keypoint 억제
- belief map 50x50 해상도에서 팔레트 높이(150mm)가 1-2px → 앞뒤 구분 원천적 어려움
- 우선순위: augmentation 강화 > geo loss 튜닝 > 해상도 증가
