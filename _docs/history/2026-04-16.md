## Phase B: Loss Ablation 재실험 — 전체 조합 (2^3)

### 학습 (3 신규 모델)

기존 A/B/C/D + 신규 F/G/H = 7 ablation 조합 + baseline (ep60).

**공통 설정** (기존 A~D 와 동일):
```
anchor  = weights/mixed_v8/final_net_epoch_0060.pth
data    = data/pallet/training_data/mixed_v8_train (9000 장)
epochs  = 65 (5 epoch ft from ep60)
lr      = 5e-5, batch = 4, sigma = 4.0
```

```
tag                λ_coord   λ_edge   λ_flip   학습시간    비고
───────────────────────────────────────────────────────────────────
F_coord_flip       0.003     0.0      0.02     1:03:20     신규
G_edge_flip        0.0       0.003    0.02     1:03:16     신규
H_all              0.003     0.002    0.02     1:03:28     신규
```

### 평가 (NN matching, middle 440 장, Hungarian, gt_final_isaac)

```
Model          Loss combination       pred#   <20px    median    <50px    비고
──────────────────────────────────────────────────────────────────────────────────
ep60 baseline  (none)                 191     18.9%    30.4px    25.2%    baseline
A              coord                  313     21.6%    66.2px    30.0%    기존
B              edge                   283     21.1%    65.3px    26.8%    기존
C ★            coord+edge             303     38.4%    16.7px    43.9%    기존 BEST
D              flip                   210     22.5%    34.0px    24.1%    기존
F              coord+flip             274     16.6%    62.9px    25.9%    신규 — 하락!
G              edge+flip              224     21.8%    47.9px    26.4%    신규
H              coord+edge+flip        216     27.3%    16.5px    34.3%    신규
```

### 해석

1. **C (coord+edge) 가 압도적 1등** — 38.4%, median 16.7px. 기존 발견 재확인.
2. **H (all three) = 27.3%** — C 보다 11pp 낮음. flip 추가가 오히려 해로움.
3. **F (coord+flip) = 16.6%** — baseline 보다 나쁨! flip 이 coord 를 상쇄.
4. **G (edge+flip) = 21.8%** — baseline 수준. edge 와 flip 상호 무효.
5. **flip 은 어떤 조합에서든 해로움**:
   - A(21.6) → F(16.6): -5.0pp (coord 에 flip 추가)
   - B(21.1) → G(21.8): +0.7pp (무효)
   - C(38.4) → H(27.3): -11.1pp (coord+edge 에 flip 추가)
6. **edge 는 coord 와 결합할 때만 유익**:
   - A(21.6) → C(38.4): +16.8pp (coord+edge synergy)
   - D(22.5) → G(21.8): -0.7pp (flip+edge, 무효)
   - 단독(B 21.1) ≈ baseline (18.9) — 의미 없음

### 논문 contribution 요약

> "coord+edge 조합이 유일한 유의미 개선 (38.4%, +16.8pp over coord-only).
> flip equivariance 는 단독/조합 모두에서 해로움 — coord loss 의 gradient
> direction 과 충돌. Edge log-ratio 는 coord 없이는 무효하지만, coord 와
> 결합 시 outlier keypoint 을 억제하여 synergistic effect."

## Phase C: F4/F5 Epoch Sweep — 학습 곡선

**목적**: "왜 31 epoch?" 질문 방어. 매 5 epoch checkpoint 전부 NN matching 평가.

```
epoch    F5 (LOO 2장)    F4 (RANSAC 9장)
─────────────────────────────────────────
ep65      21.6%           21.6%          (= CoordDOPE anchor)
ep70      20.9%           28.9%  ← F4 peak
ep75      39.8%           25.2%
ep80      47.7%           25.9%
ep85      55.5%           26.4%
ep90      58.2%           21.8%
ep95      60.0%           13.6%
ep96      60.5%  ★        14.3%
```

**핵심 관찰**:
1. **F5 단조 증가** — ep70 20.9% → ep96 60.5%, 수렴 미완성 (slope 여전히 양)
2. **F4 ep70 정점 후 하락** — 28.9% → 14.3% (overfitting)
3. **교차점 ep70** — 그 이전에는 F4 > F5, 이후 F5 급상승
4. 31 epoch은 arbitrary 했으나 결과적으로 F5에게는 **lower bound** (더 돌리면 더 올라감)

**방어 답변**: "동일 학습 예산(31 epoch)에서 LOO 2장은 단조 개선, RANSAC 9장은
과적합. 이 divergence 자체가 필터 품질의 증거. F5는 수렴 전이라 60.5%는 하한."

**원본**: `data/pallet/eval_results/f4_f5_epoch_sweep.txt`

## Phase A-4: F4 epoch 결정

**쟁점**: F4 표에 ep90 (21.8%) 쓸지 ep96 (14.3%) 쓸지. F5 는 ep96 (60.5%).

**판단**: **ep96 통일 권장 (옵션 3)**.

**이유**:
1. **공정 비교** — F4/F5 모두 동일 ft setting, 동일 anchor (ep65), 동일 total
   epochs (96). 서로 다른 epoch 끊기는 cherry-pick 의심.
2. **이야기가 더 강해진다** — "같은 31 epoch 학습에서 RANSAC-only 는 overfit
   (21.8→14.3), RANSAC+LOO 는 계속 개선 (58.2→60.5). LOO 필터의 training
   stability 기여를 직접 보여주는 증거."
3. **질문 방어** — "왜 F4 만 ep90 썼어?" committee 질문 차단.

**권장 표 (최종)**:
```
Method                                  Filter           PL#   <20px   median
──────────────────────────────────────────────────────────────────────────────
Synthetic-only baseline (ep60)          —                —     18.9%   30.4px
+ CoordDOPE ft (ep65)                   —                —     21.6%   66.2px
ST_8only (real-only 91 ep)              canonical B∧C     8    37.3%   33.5px
F4 RANSAC-only (real-only ep96)         RANSAC c≥6        9    14.3%   —
F5 RANSAC+LOO (real-only ep96) ★        RANSAC+LOO        2    60.5%   12.2px
```

**해석 문장 (발표용)**:
> "동일 31 epoch 학습에서 RANSAC-only 는 overfitting (21.8% → 14.3%), LOO
> 추가 시 안정적 개선 (58.2% → 60.5%). 필터 품질이 training dynamics 자체를
> 결정한다."

※ 만약 Phase A 숫자 확인 후 "ep90 21.8% 쪽이 더 나은 발표 내러티브" 로
최종 판단되면 옵션 1 (F4 ep90, F5 ep96) 로 복귀 가능. 단 옵션 3 이 공정성
