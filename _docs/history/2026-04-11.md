## One-Sided Collapse 메커니즘 정리 (failure mode 기록)

v8_A 추론에서 한쪽에 keypoint가 몰려 PnP가 실패하는 케이스를 관찰. 이건 단순 detector 노이즈가 아니라 DOPE belief-map 학습 구조의 근본적 한계를 드러내는 failure mode이므로 기록.

### 관찰
- Real (capture0403middle) 일부 프레임에서 9/9 또는 6/9 keypoint가 검출되었음에도 PnP fail
- 검출된 점이 파렛트 한쪽 절반(좌 또는 우)에 몰림 — 반대쪽은 peak 자체가 약하거나 잘못된 위치에 생성
- 합성 데이터에서는 발생하지 않음 → train/test domain gap의 직접적 증거

### 메커니즘
1. **Belief map이 한쪽에서만 peak를 만듦**
   - 저대비/유사 색상 영역에서 belief activation이 약함
   - DOPE의 pixel-wise MSE 학습은 local supervision만 제공 → 반대쪽 peak를 보강하는 context 신호 없음

2. **soft-argmax는 잘못된 peak를 그대로 좌표로 변환**
   - peak가 약하거나 없으면 soft-argmax는 그 채널의 잔여 activation으로부터 임의 위치를 뽑음
   - 결과: "확실하게 잘못된" 좌표가 출력됨

3. **Structural loss(VP / edge / flip)는 이 좌표를 고치지 못함**
   - structural loss는 soft-argmax 출력 좌표 위에 걸림
   - 좌표를 backprop으로 밀어도 belief map MSE의 gradient가 압도적으로 지배 → belief activation은 거의 안 움직임
   - 즉 structural loss는 "이미 잘못된 peak 위치"를 받아쓰고만 있으며, peak 자체를 만들거나 옮기지 못함
   - 이게 v8_B/B2/B3/D/VP가 모두 실패한 통합 원인 (`project_structural_loss_line_closed.md`)

4. **Belief map 자체를 바꾸는 건 belief MSE 뿐**
   - 따라서 "안 보이는 쪽에 peak를 만들게 하는" 학습 신호는 belief target 자체에서 나와야 함
   - loss 추가가 아니라 **데이터(DR 강화, real self-training) + GT 재정의(visibility-aware sigma)** 만 이 문제를 풀 수 있음

### 함의
- **Loss line은 종료**: structural regularization으로 one-sided collapse를 잡는 시도는 더 이상 하지 않는다
- **3개 축에서만 해결 가능**:
  1. **데이터**: DR 강화 (저대비/가림 케이스 추가) + self-training으로 real 적응 (이미 49.5%→78.7% 입증)
  2. **GT 재정의**: visibility-aware sigma — visible=4.0 / self-occluded=6.0 / invisible=0 → belief MSE가 보이는 쪽에 집중하고 반대쪽 false peak 억제
  3. **Geometric Filter**: 발생한 one-sided collapse를 pseudo-label에서 걸러냄 — Filter B (Endpoint Support, Span Ratio)가 정확히 이 mode를 잡도록 설계됨
- 이 구조가 본 연구의 contribution을 정당화: detector를 고치는 대신 (1) 데이터로 적응하고 (2) 출력은 필터로 검증

### 논문 활용
- Failure mode 시각화 자료로 사용 가능 — "9개 다 검출됐는데 PnP 실패"는 단순 detection failure가 아니라 정확히 이 메커니즘을 보여주는 강한 예시
- Filter B 존재 이유의 직접적 motivation 그림으로 사용

## Filter Design Rationale 정리

위 P/R 결과와 이미 확립된 loss line-closed 분석을 합쳐, "왜 RANSAC 하나로
충분한가" 를 별도 문서로 박제 — `_docs/filter/2026-04-11_design_rationale.md`.

핵심 주장:
1. **self-training 은 기하학적 이해를 주지 못한다** — CNN 구조 (flip equivariance,
   padding 비대칭) 는 데이터 적응으로 바꿀 수 없음. 근데 **해결할 필요가 없음**:
   belief MSE 가 암묵적으로 cuboid 구조를 복사하므로 남은 오차는 "구조 붕괴"가
   아니라 "위치 밀림" (VP/edge/flip loss dead-on-arrival 결과와 일치).
2. **역할 분담이 이미 성립**:
   - 기하학적 구조 유지 → belief MSE (자동)
   - 위치 정밀도 → coord loss
   - domain 적응 → self-training (49.5%→78.7% 입증)
   - outlier 제거 → RANSAC filter (F1=0.833 입증)
   - flip equivariance → 필요 없음
3. **필터 구조**: pre-filter (detected≥5) → main gate (RANSAC c≥6, decision
   power 95%) → post-check (size sanity). 3단계지만 사실상 RANSAC 하나.
4. **더 쌓아도 좋아지지 않음**: B∧C 를 RANSAC 위에 추가하면 오히려 악화 (F13
   실측 확인).
5. **논문 프레이밍**: "필터 1개" 는 약점이 아니라 "23 후보를 엄밀히 비교 끝에
   남은 결과" 로 contribution. negative result (canonical B∧C 실패) 도 포함.
