## 2. 멀티소스 학습 실험 3종

### 데이터 분할 및 학습

```
실험            Train 데이터                            Train 크기    Val 크기
──────────────────────────────────────────────────────────────────────────────
blender_v1      Blender only                                 3,600        400
combined_v1     Isaac Sim 6K + Blender 3.6K                  9,600      1,900
mixed_v1        Isaac 기본 2K + far 2K + Blender 4K (1:1)    8,000        800
```

- blender_train_v1 (4000장)을 90/10으로 split (seed=42)
- combined: Isaac Sim train 6000 + blender 3600, 번호 재부여
- mixed: Isaac 기본 2000(샘플링) + far 2000(전수) + Blender 4000, val도 비율 맞춤

### 공정 재평가 (mixed_v1_val 800장, max_frames=800)
- **Before**: 각 실험별 val에서 max_frames=200 → source 편향 발생 (combined_v1이 과소평가됨)
- **After**: 3개 모델을 동일한 source-balanced val 800장에서 재평가

```
메트릭          blender_v1    combined_v1    mixed_v1
──────────────────────────────────────────────────────────────────────────────
PCK@3px              0.540          0.570       0.593 *
PnP Rate             70.8%          94.4%       91.1%
Reproj mean         83.0px         67.3px      53.4px *
Vol<20%              50.1%          53.4%       54.7% *
```

- **결론**: mixed_v1이 PCK + Reproj 1위. 이전 편향 평가에서 blender_v1 > combined_v1이었던 것이 역전.

## 6. output_size 불일치 정리

### Shape 확인
- `train.py`는 `output_size = 50` 하드코딩
- 실제 runtime: GT (4, 9, **50**, **50**) / Network output (4, 9, **50**, **50**) → 일치
- config/default.yaml과 stage3_selftrain.yaml이 56으로 잘못 적혀 있었음 (config drift)

### 수정
- **`config/default.yaml`**: output_size 56 → **50**
- **`config/stage3_selftrain.yaml`**: belief_map_size 56 → **50**
- **`Deep_Object_Pose/train/train.py`**: epoch 0 batch 0에서 shape assert 추가
- **기존 weight 영향**: 없음 (runtime은 원래부터 50×50 정상)
- **기술부채**: train.py의 하드코딩 → 여유 시 config에서 읽도록 정리 필요

## 10. Stage 3 Smoke Test — End-to-End 파이프라인 검증

### 결과
- **Step 1 (Inference)**: PnP OK 18/20 (90%) — mixed_v1 모델 정상
- **Step 2 (Filter)**: legacy filter로 4/20 통과 (acceptance 20%)
- **Step 3 (Pseudo-label)**: 4장 PNG + JSON 저장, DOPE DataLoader 호환 확인
- **Step 4 (Finetune)**: synthetic 50 + pseudo 4 = 54장, loss 0.033→0.039, NaN 없음

### 발견된 이슈 및 수정
1. **Legacy C2 (cuboid geometry) 구조적 문제**: front-to-rear edge angle이 정면뷰에서 170°+로 정상인데 tau_angle_max=120/135에 걸림. threshold 완화(10~175°)로 우회, 근본 해결은 새 필터(Diagonal Concurrency)로 교체 필요
2. **Pseudo-label 저장 포맷**: `.jpg` → `.png` 변환 필요 (DOPE DataLoader는 `{i:06d}.png` 기대)
3. **Pseudo-label JSON**: `visibility` 필드 누락 → 추가 (DataLoader KeyError 해결)

### 사용된 threshold (legacy_filter_smoke_v1)
```
tau_reproj: 12.0, tau_ratio: 0.4~2.5, tau_angle: 10~175°, tau_size: 0.5~2.5m
```

- `scripts/self_training/smoke_test.py` — inference → filter → pseudo-label → finetune 1 epoch
