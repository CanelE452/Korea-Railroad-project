## On-the-fly truncation augmentation (DOPE 로더 내장)

challenge pretrain용 offline 고정 crop(2x, 매 epoch 같은 잘린 픽셀→암기 과적합)을
로더가 매 샘플 확률적으로 truncation crop+pad 하는 방식으로 대체. 데이터 1배 유지
(시간 절반), 매 epoch 다른 random crop → 잘림 다양성↑.

### 변경 파일
- `Deep_Object_Pose/common/utils_dataset.py`
  - offline `challenge/scripts/gen_truncation_crops.py`(crop+resize 640x480,
    CUT_WEIGHTS L/R 편향, degenerate reject) + `pad_truncation_crops.py`
    (reflect-pad off-img kp back, MARGIN_FRAC=0.20)를 in-memory numpy 함수로 포팅
    → module-level `apply_truncation_aug(img, kps9, rng)` 및 `_trunc_*` 헬퍼.
  - `CleanVisiiDopeLoader.__init__`에 `truncation_aug_prob=0.0` 인자(기본 off,
    backward compat). `import cv2`, `import random` 추가.
  - `__getitem__`: raw 로드 직후 **albumentations 전에** Bernoulli(prob) 적용.
    단일 obj + 유효 9kp만 대상. rng = worker_info.seed ^ (index*hash)
    → epoch/worker/sample마다 다른 crop.
  - **RandomCrop 상호작용 처리**: trunc 샘플은 `A.RandomCrop(400,400)`을
    `A.Resize(400,400)`로 교체(이미 truncate된 팔레트 재절단 방지, padding으로
    넣은 corner 보존). clean 샘플 경로 완전 불변. Rotate/색조 augs는 양쪽 유지.
- `Deep_Object_Pose/train/train.py`: `--truncation_aug_prob`(default 0.0) argparse
  추가 + 로더 전달.
- `scripts/train_dope.sh`: `--truncation_aug_prob`(default 0) passthrough + info echo.
- `challenge/scripts/_verify_truncation_aug_loader.py`(신규): 검증 스크립트.

### 검증 (env pallet-pose, N=200 v1/v2 19968장)
- prob=0.6: truncation 56% / clean 44% (Bernoulli, 목표 60:40 근사).
- prob=1.0 강제: 191/200 success(9 = degenerate retry-exhausted 정상),
  belief 50x50 supervised median=9, 9/9 비율 95.3%(나머지는 Rotate aug lottery).
- **순수 geometry(rotate/crop 빼고 CreateBeliefMap 직접): 189/189 = 100% 9/9**
  → crop+pad+MARGIN0.20 자체는 완벽. 잘렸던 corner도 전부 supervised.
- prob=0: truncation 적용 0건 (기존 학습 동작 불변 확인).
- overlay: `challenge/data/_verify_truncation_aug/trunc_*.png` (reflect-pad 타일링 +
  9코너 wireframe 전부 belief peak).
- convention: crop/pad 모두 pure affine → projected_cuboid 순서 보존(camera-facing
  v4). flip 없음.

### pretrain 실행 (train_dirs = v1/v2 원본만, on-the-fly로 trunc 생성)
```
bash scripts/train_dope.sh --exp_name challenge_pretrain_otftrunc \
  --train_dirs "challenge/data/training/v1/part_*/train_palletobj_v1 challenge/data/training/v2/part_*/train_palletobj_v2" \
  --truncation_aug_prob 0.6
```

## FSM에 YOLO pose backend 추가 (DOPE 전용 → DOPE/YOLO 선택)

기존 lifter 정렬 FSM(`25y_automatic_lifter-master/.../depth_cam/`)은 DOPE pose만 받게 돼 있었음. YOLO26-pose도 동일 FSM에 연결.

### 신규/변경
- **신규** `depth_cam/calib/yolo_inference.py` — `YoloPoseEstimator`. `DopePoseEstimator.infer_pose(bgr,K,depth)`와 동일 계약(dict: R, t_m, confirmed, gate_passed, raw/proj_points_orig, kps9_orig, info, yaw_deg_cam). YOLO+100px pad+shift+SQPnP(make_pallet_keypoints_3d, camera-facing 0123)+sanity gate+N연속 confirm. weights는 .pt/.onnx/.engine.
- **★ convention fix**: DOPE Cuboid3d는 front=+Z, YOLO make_pallet_keypoints_3d는 near/front=−Z (far=+Z) → +Z 부호 반대(Y축 180° 차이). pose6d_adapter는 DOPE(front=+Z) 기준이라 **R_fix=diag(-1,1,-1) (Ry180)** 적용: `R_dope = R_yolo @ R_fix`. t는 centroid라 불변. 미적용 시 정면 ψ≈±179°(오), 적용 시 ψ≈0(정상) 실측 확인.
- **변경** `depth_cam/calib/config.py` — `POSE_BACKEND`("dope"|"yolo", env override), `MODEL_PATH_6D_YOLO`(기본 pallet_jetson_deploy/models/pallet_pose_cropaug_v2.pt).

## Jetson 포크 삽입 준비 — depth fusion + real-mode + Jetson 패키지
